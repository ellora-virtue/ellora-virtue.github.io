HTML and CSS - an analogy that may or may not be original...

I’ve been racking my brain trying to come up with an original analogy to explain HTML, CSS and how they work together… So I’m going to use a cake analogy.

Imagine a beautifully decorated, three-tier cake. In this analogy, HTML is the cake itself, made up of a blend of different ingredients to form the base. HTML might also define some of the decorations, for example the cake might have a photo on the top. CSS comes in to control how the cake is decorated, or “styled”. CSS would decide what shape each layer of the cake is, how high, what colour the icing is, where the decorations are placed, and so on.

Control flow and loops

Control flow is the order JavaScript is executed by the browser. This is important to recognise and make use of, as often times it may be causing a bug in your code or altering the way things should be running due to being in the wrong order.

Loops are a very useful part of JavaScript designed to allow you to repeat the same code over and over again until a certain condition is met. To save you writing out super repetitive code, there are three main looping options: for loops, while loops and do while loops.

For loops are the most commonly used and take three optional expressions in the head of the loop, along with a statement that is executed as long as the conditions are equal to true. What is the DOM?

The DOM stands for the Document Object Model. It lives mostly behind the scenes, is made up of a tree of objects called “nodes” and looks pretty similar to a family tree (except each node only has one parent), as you can see here:

It starts at the top with your HTML document and then branches off to different sections of the website (e.g. head and body). Each of these sections then have various elements branching off them (their children - div, h2, p etc.) with different values. These can also have further children branching off them, and so on.

When you load a web page, the browser takes the HTML from wherever it may be stored and converts it into the DOM. From there, we can use JavaScript to manipulate HTML elements and add, change or remove things from our page.

The DOM could be very simple, as we see in the above diagram, or much more intricate and complicated-looking. In saying that, if you dive deep into each section of the DOM, it is not as difficult or complicated as it looks!

You can access the DOM from any webpage by right-clicking and selecting “Inspect”, which brings you to Google’s Developer Tools.

Accessing data from arrays and objects

Arrays and objects are two of JavaScript’s data types.

An array is a collection of data used to store multiple values (as opposed to a variable which can only store one value). Each item in an array is contained within square brackets [ ], separated by a comma and given a number index (the first item starting at 0, second item at 1, and so on) in order to be easily accessible. A brief overview of functions

Functions are brilliant for writing an executable piece of code that has a function and can be used over and over again throughout your document.

It keeps code cleaner and tidier by not repeating the same thing again and again, and enables you to do all sorts of interesting things with JavaScript. (Note: you must define a function within the scope you wish to call it, e.g. global or local scope, otherwise it will not run.)

To define a function, the function keyword is used, followed by the function name (can be anything), a set of parentheses containing a list of parameters to be used by the function, and finally the statements that the function will execute surrounded by curly brackets - { }.
